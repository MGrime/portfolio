Multi-Threaded Collision Simulator (C++)

Project Overview

Source Code - MGrime/Multithreading-Visualiser: Simulates collisions between millions of circles efficiently. (github.com)

Project Goal - To create a multithreaded program to simulate collision between millions of circles in an effective timescale.

Concepts Used - Multithreading, Data-Oriented-Design (DoD), Macros, Optimisation, Linear Algebra

Project Technologies

Tools - GitHub / Visual Studio

Languages - C++

Libraries - Eigen, TL-Engine, SDL (Subsystems)

Time Allowed - 2 weeks

Discussion

This project is aimed to let me push multithreading and optimisation as far as I can.

The program is simulating a scenario in which a number of moving circles are moving and colliding with a number of stationary circles. When a moving circle collides with a stationary circle it bounces off that circle and continues in its new velocity.

My current implementation of this simulation uses multithreading and DoD to allow for simulation of circle counts in the millions in a reasonable timeframe.

The program will use as many threads as it can, and the workload scales very effectively across multiple cores, making it act as an effective stress test (See Screenshot 3 below). The program can be configured to push even a Ryzen 7 5800X (8 core/16 thread) to 100% utilisation.

DoD has been applied to how the data is stored for the circles, with data being split into separate arrays of similar data. For example all of the positions and radiuses for the stationary circles are in one array, same for the moving circles array. This allows the program to be cache efficient as all required data when resolving collisions is next to each other in memory.

Configuration of the program is done by changing pre-processor macros and constants in the source code, however this is a limitation to fit the assignment brief. I eventually plan to create a configurator program to make it easier to run on other hardware. However in it's current state changing the configuration requires you to change the source code and recompile the project.
